Program Structure
As of now, we aim to make the student community familiar with the APIs and build their own application. So we have scheduled discussions on it, about some common and best APIs. We are planning to come up within this month. As the initial stage, we are planning to this month. Further, it can be expanded to alternatives Saturdays in a month. A repository will be created for this respectively, in which a subfolder will be allocated for the API that is to be conducted on that particular day. The folder will be shared with the registered participants. Thus after completion of their application, they can commit their changes to the folder. The procedure finalized till now is

Process Flow
step 1: We will fix the API and share some clues about the API in social media networks to guess it.
step 2: 2 days before the Saturday hack night we will publish the API and the documentation about it and some problem statements also.
step 3: Registration begins and participants can be registered as a group
Hack night day
Step 4: We will explain the API documentation and address the queries around it
Step 5: share the Repo link and mention the team leads to create a folder with the team name, commit it has a draft
step 6: teams can start work on the problem statement
step 7: 2 to 2:30 hrs after the program kick starts, people can pitch about their problem statements and queries about the project
step 8: team can host the project and mention the link in the last commit message.
Step 9: our internal team can evaluate the code structure and publish the best projects.
